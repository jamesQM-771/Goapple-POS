
Perfecto, ahora sí te voy a dejar un **flujo profesional paso a paso** para tu día a día con el proyecto GOAPPLE 🚀
Esto es básicamente cómo trabaja un desarrollador real, pero adaptado a ti.

---

# 🔄 🧠 FLUJO COMPLETO (CADA VEZ QUE TRABAJES)

## 🟢 1. Abrir tu entorno

Cada vez que vayas a trabajar:

1. Abre **Laragon**
2. Inicia:

   * Apache
   * MySQL
3. Abre tu IDE (Antigravity)
4. Abre tu proyecto:

   ```
   C:\laragon\www\goapple
   ```

---

## 🔄 2. Sincronizar con GitHub (IMPORTANTE)

Antes de tocar código:

```bash
git pull
```

👉 Esto asegura que tienes la última versión
(Si trabajas solo, igual hazlo por costumbre)

---

## 🧪 3. Ejecutar el proyecto

* Abre en navegador:

  ```
  http://localhost/goapple
  ```
* Verifica que todo funcione

👉 Esto evita trabajar sobre algo roto

---

## ✏️ 4. Hacer cambios

Aquí haces lo tuyo:

* código PHP
* vistas
* base de datos (HeidiSQL)
* pruebas

💡 Ejemplos:

* arreglar errores
* mejorar diseño
* agregar funcionalidades

---

## 🧪 5. Probar TODO

Antes de guardar cambios en Git:

✔ prueba lo que hiciste
✔ prueba lo que ya existía
✔ evita romper cosas

👉 Este paso es CLAVE (muchos lo saltan)

---

## 🔍 6. Revisar cambios

```bash
git status
```

👉 Te muestra qué modificaste

---

## 💾 7. Guardar cambios (commit)

```bash
git add .
git commit -m "mensaje claro"
```

---

## 🧠 Ejemplos de mensajes reales:

```bash
git commit -m "fix: error al eliminar cliente"
git commit -m "feat: agregar campo notas"
git commit -m "update: mejoras en formulario clientes"
```

---

## ☁️ 8. Subir a GitHub

```bash
git push
```

👉 Ya queda guardado en la nube

---

# 🔁 🔁 RESUMEN RÁPIDO (TU RUTINA)

```bash
git pull
# trabajas
git add .
git commit -m "mensaje"
git push
```

---

# 🧠 CONSEJOS IMPORTANTES (DE VERDAD)

## 🔥 1. No trabajes sin hacer pull

Evita conflictos después

---

## 🔥 2. No subas código roto

Siempre prueba antes

---

## 🔥 3. No hagas commits basura

❌ `"cambios"`
❌ `"prueba"`

✔ Sé claro

---

## 🔥 4. Haz commits por bloques

NO:

* un commit gigante con todo

SÍ:

* uno por cada avance importante

---

# 🧪 EJEMPLO REAL DE TU DÍA

Hoy hiciste:

* arreglaste eliminar cliente
* mejoraste formulario

Entonces:

```bash
git pull
git add .
git commit -m "fix: eliminacion cliente y mejoras formulario"
git push
```

---

# 🧯 🚨 SI ALGO SALE MAL

## ❌ Me equivoqué en un commit

```bash
git reset --soft HEAD~1
```

---

## ❌ Rompí algo

Puedes volver atrás (luego te enseño bien esto si quieres 👀)

---

👉 trabajar con **ramas (branch)**
👉 hacer pruebas sin dañar tu sistema principal
